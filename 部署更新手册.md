# 沪深300多因子选股系统 - 工程更新部署手册

## 一、更新内容概述

本次更新为沪深300多因子选股系统的增量细化升级，主要包含以下内容：

1. **量化核心升级**：
   - 支持2019-2026年数据范围
   - 新增分行业因子分析（IC/IR值、单调性检验）
   - XGBoost参数调优（学习率、树数量组合）
   - 多场景回测（牛市/熊市/震荡市、月度/季度调仓）
   - 完整风险指标体系
   - 实验配置文件（YAML格式）
   - 自动运行日志

2. **后端API增强**：
   - 新增因子分行业分析接口
   - 新增风险指标详情接口
   - 新增模型调优接口
   - 新增多场景回测接口
   - 新增实验日志接口
   - 统一响应格式和参数筛选

3. **前端可视化升级**：
   - 风险指标详情页（卡片式展示、雷达图）
   - 模型参数调优页（热力图、性能对比）
   - 因子分行业分析页（柱状图、数据表格）
   - 所有图表支持导出PNG
   - 数据表格支持导出CSV
   - 响应式布局优化

## 二、部署环境要求

### 2.1 硬件要求
- CPU: Intel Core i5或更高

- **Node.js**: 16.x或18.x

## 四、数据更新步骤

### 4.1 数据文件说明

本次更新使用模拟数据，无需外部数据源，系统会自动生成所需数据文件：

- `data/hs300_constituents.csv` - 沪深300成分股数据
- `data/training_data.csv` - 训练数据
- `data/processed_data.csv` - 处理后的数据
- `data/prediction_results.csv` - 预测结果
- `data/industry_factor_analysis.csv` - 分行业因子分析数据
- `data/model_parameter_results.csv` - 模型参数调优结果
- `data/backtest_results_advanced.csv` - 多场景回测结果
- `data/risk_metrics.csv` - 风险指标数据
- `data/data_quality_report.json` - 数据质量报告

### 4.2 数据生成流程

1. **运行量化核心模块**（详见第五章），系统会自动生成所有数据文件
2. 数据生成完成后，会在控制台显示数据文件保存路径
3. 数据生成时间约为1-2分钟，取决于硬件性能

## 八、常见问题与解决方案

### 8.1 依赖安装问题

**问题**：Python依赖安装失败
**解决方案**：
- 确保Python版本为3.9
- 使用国内镜像源：`pip install -r quant_core\requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`
- 检查网络连接

**问题**：Node.js依赖安装失败
**解决方案**：
- 确保Node.js版本为16.x或18.x
- 使用国内镜像源：`npm install --registry=https://registry.npmmirror.com`
- 清理缓存：`npm cache clean --force`

### 8.2 服务启动问题

**问题**：后端服务启动失败
**解决方案**：
- 检查端口8000是否被占用
- 检查Python依赖是否安装完整
- 查看控制台错误信息

**问题**：前端服务启动失败
**解决方案**：
- 检查端口5173是否被占用
- 检查Node.js依赖是否安装完整
- 查看控制台错误信息

### 8.3 数据加载问题

**问题**：前端数据加载失败
**解决方案**：
- 检查后端服务是否正常运行
- 检查网络连接
- 检查浏览器控制台错误信息
- 重新运行量化核心生成数据

**问题**：量化核心运行失败
**解决方案**：
- 检查Python依赖是否正确
- 检查磁盘空间是否充足
- 查看运行日志了解具体错误

### 8.4 图表渲染问题

**问题**：ECharts图表不显示
**解决方案**：
- 检查浏览器是否支持Canvas
- 检查数据是否正确加载
- 检查控制台是否有JavaScript错误

**问题**：图表导出失败
**解决方案**：
- 检查浏览器权限设置
- 检查图表数据是否完整
- 尝试使用Chrome浏览器

## 九、故障排除流程

1. **检查系统状态**：访问 `http://localhost:8000` 检查后端服务状态
2. **查看运行日志**：检查 `logs/` 目录下的运行日志
3. **检查数据文件**：验证 `data/` 目录是否存在所有必要数据文件
4. **检查依赖状态**：重新安装依赖包
5. **重启服务**：按顺序重启量化核心、后端服务、前端服务
6. **清除缓存**：清除浏览器缓存，重新加载页面
7. **查看错误信息**：分析控制台和日志中的错误信息
8. **寻求帮助**：如果问题持续存在，请联系技术支持

## 十、更新注意事项

1. **数据备份**：在更新前建议备份 `data/` 目录下的所有数据文件
2. **依赖版本**：严格按照指定版本安装依赖，避免版本冲突
3. **端口占用**：确保8000（后端）和5173（前端）端口未被占用
4. **浏览器兼容性**：推荐使用Chrome浏览器获得最佳体验
5. **性能优化**：对于低配置机器，可适当减少数据生成规模
6. **安全注意事项**：本系统仅用于本地部署，不要在公共网络暴露服务
